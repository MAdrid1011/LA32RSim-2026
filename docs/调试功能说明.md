# 调试功能说明

## 概述

LA32RSim-2026 提供多种调试功能，均可通过 Kconfig 独立开关。所有功能在 `make menuconfig` 的「调试与追踪」菜单中配置。

---

## 1. 指令执行追踪（ITRACE）

**配置项：** `CONFIG_ITRACE=y`，`CONFIG_ITRACE_SIZE=32`

ITRACE 维护一个环形缓冲区，记录最近 N 条提交指令的信息。

**记录内容：**
- 提交周期号
- 指令 PC
- 指令编码（反汇编显示）
- 是否写寄存器、写哪个寄存器、写入的值

**触发时机：** 仿真失败（difftest 不一致、BAD TRAP、死锁）时自动打印。

**示例输出：**
```
── 最近 32 条指令追踪（ITRACE）──
  周期      PC          指令                                     写入值         目标寄存器
  ─────────────────────────────────────────────────────────────────────────────
  1001      0x1c000000  addi.w   r3, r0, 0x100                  0x00000100     r3
  1002      0x1c000004  ori      r4, r0, 0xa5                   0x000000a5     r4
  1003      0x1c000008  st.w     r4, r3, 0                      -              -
  1004      0x1c00000c  ld.w     r5, r3, 0                      0x000000a5     r5
  ...
```

---

## 2. 内存访问追踪（MTRACE）

**配置项：** `CONFIG_MTRACE=y`

每次内存读写都打印一行日志。输出量大，建议仅在调试内存相关问题时开启。

**AXI 模式输出：**
```
[mtrace] cycle=1003     AXI-W addr=0x1c000100 data=0x000000a5 wmask=0xf
[mtrace] cycle=1004     AXI-R addr=0x1c000100 data=0x000000a5
```

**简单总线模式输出：**
```
[mtrace] cycle=1003     WRITE addr=0x1c000100 data=0x000000a5 wmask=0xf
[mtrace] cycle=1004     READ  addr=0x1c000100 data=0x000000a5
```

---

## 3. 内存写入时间戳追踪（MEM_WTRACE）★

**配置项：** `CONFIG_MEM_WTRACE=y`

以**字节粒度**为每个内存字节维护「最后一次写入周期」表。

### 工作原理

每次内存写操作时，根据字节使能（`wmask` / `wstrb`）逐字节记录当前周期号：

```
st.w r4, r3, 0     # 写4字节，周期号 = 1003
  → 地址 0x1c000100 字节 0: 最后写入周期 = 1003
  → 地址 0x1c000101 字节 1: 最后写入周期 = 1003
  → 地址 0x1c000102 字节 2: 最后写入周期 = 1003
  → 地址 0x1c000103 字节 3: 最后写入周期 = 1003

st.b r5, r3, 2     # 写1字节（字节2），周期号 = 2007
  → 地址 0x1c000102 字节 2: 最后写入周期 = 2007  （更新）
  → 其他字节保持 1003                              （不变）
```

### 触发时机

Difftest 失败时，自动打印提交 PC 附近内存的写入时间戳：

```
── 内存写入时间戳（字节粒度）──
  内存写入时间戳 (addr=0x1c000100):
  0x1c000100:  [cyc1003 ]  [cyc1003 ]  [cyc2007 ]  [cyc1003 ]
  0x1c000104:  [cyc999  ]  [never   ]  [never   ]  [cyc999  ]
  0x1c000108:  [never   ]  [never   ]  [never   ]  [never   ]
  0x1c00010c:  [cyc1100 ]  [cyc1100 ]  [cyc1100 ]  [cyc1100 ]
```

### 主动查询

在仿真代码中可调用：

```cpp
// 查询单字节最后写入周期（返回 -1 表示从未写入）
int64_t cyc = mem->queryWriteCycle(0x1c000102);

// 打印一段地址范围的写入时间戳（nwords 个字）
mem->dumpWriteTimestamp(0x1c000100, 4);
```

### 配合波形调试的使用方法

1. 通过 MTRACE 发现某地址数据异常
2. 用 MEM_WTRACE 确认该地址最后写入是哪个周期（例如周期 2007）
3. 在 Kconfig 中设置 `WAVE_BEGIN=2000, WAVE_END=2010`，重新运行
4. 用 GTKWave 打开 `build/waveform.vcd`，精确定位问题

## 4. VCD 波形生成（DUMP_WAVE）

**配置项：** `CONFIG_DUMP_WAVE=y`

Verilator 会在仿真过程中生成 `build/waveform.vcd`。

**周期范围控制（避免文件过大）：**

```
CONFIG_WAVE_BEGIN=10000   # 从第 10000 个周期开始录制
CONFIG_WAVE_END=11000     # 在第 11000 个周期停止录制（0=不限制）
```

**配合 MEM_WTRACE 快速定位：**

```bash
# 1. 先运行一遍不录波形，获取失败时打印的周期号
make run IMG=test.bin

# 2. 根据输出的周期号设置范围，重新配置
make menuconfig
# 设置 WAVE_BEGIN=<失败周期-100>, WAVE_END=<失败周期+10>

# 3. 重新编译运行
make all
make run IMG=test.bin

# 4. 用 GTKWave 查看
gtkwave build/waveform.vcd
```

## 5. 进度显示

仿真过程中，后台线程每秒打印当前状态（无需配置，始终开启）：

```
[进度] 周期: 1234567    提交指令: 987654     IPC: 0.800
```

---

## 6. 彩色输出（DEBUG_COLORS）

**配置项：** `CONFIG_DEBUG_COLORS=y`

- GOOD TRAP → 绿色
- BAD TRAP / 错误 → 红色
- 警告 → 黄色
- 差异寄存器 → 红色高亮

---

## 7. UART 输出（UART_STDOUT）

**配置项：** `CONFIG_UART_STDOUT=y`

将 MMIO UART 的写操作输出到 stdout，模拟串口。
测试程序中的 `printf` 等可通过 UART 输出文字。

**支持的 UART 地址：**

| 地址 | 来源 |
|------|------|
| `0xa00003f8` | base-port 运行时（`SERIAL_PORT`），functest/coremark/dhrystone/picotest 均使用此地址 |
| `0x1FE001E0` | 框架默认 UART 地址（`UART_ADDR`） |

两个地址均被框架自动识别，无需额外配置。

## 调试流程推荐

```
出现 difftest 不一致
       │
       ▼
查看 ITRACE 最近指令历史
       │
       ├─ 找到可疑指令 → 检查 MEM_WTRACE（该地址最后写入周期）
       │                          │
       │                          ▼
       │                 设置 WAVE_BEGIN/END → 重新运行 → GTKWave
       │
       └─ 找不到规律 → 开启 MTRACE（记录所有内存访问）
                               │
                               ▼
                      缩小范围 → 关闭 MTRACE + 设置 WAVE 范围
```
